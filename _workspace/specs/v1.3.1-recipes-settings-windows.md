Each window is an `NSWindow` with a single `WKWebView` content view,
lazy-created on first open and retained for the session. Re-clicks
focus the existing window. Close the window ‚Üí it stays in memory but
is hidden; next click re-shows + refreshes data.

## Recipes data model

**Stock recipes** ‚Äî hard-coded in `core/recipes.py::STOCK_RECIPES`:

```python
{
    "id":          "stock-zoom-notes",      # stable id; "stock-" prefix
    "name":        "Zoom + notes side by side",
    "category":    "Meetings",
    "description": "Opens the meeting on the left half, Notes on the right.",
    "body":        "+CalFlow+\nopen zoom.us @chrome #left(50%)\n‚Ä¶",
    "icon":        "users",                  # tabler-icon hint for UI
}
```

8 starters across categories: Meetings (2), Focus (2), Daily (2), Multi-monitor (2).

**My recipes** ‚Äî `data/my_recipes.json`:

```json
{
  "schema_version": 1,
  "recipes": [
    {"id": "mine-XXXX", "name": "...", "category": "Meetings",
     "body": "...", "created_at": "...", "updated_at": "..."}
  ]
}
```

`id` is `"mine-" + uuid4().hex[:8]`. Updates are upsert by id.

## Settings ‚Äî read-only contract

`cli.main settings --json` returns the human-readable subset of
`config/settings.py` grouped by section. Hidden fields:
`AUTOFILL_SHORTCUTS` (long dict), `BLACKLIST_REGEX` (security),
`MAP_DOMAINS` (large list ‚Äî surface count only).

```json
{
  "general":  {"auto_start_at_login": true, "show_menu_bar_icon": true, ...},
  "calendar": {"google_account": "breezm‚Ä¶", "calendars_watched": "3 of 7"},
  "events":   {"open_minutes_early": 5, "default_browser": "Chrome", ...},
  "advanced": {"trigger_grace_seconds": 600, "lookahead_window_min": 30,
               "blocked_url_patterns": "10 rules"}
}
```

The settings page **does not write** in v1.3.1. Each section has an
"Edit in settings.py" button that opens the file in the user's default
editor (`open -t config/settings.py`). Native form-based editing is
deferred to v1.4.x ‚Äî needs a settings JSON refactor first.

## Run-script safety

`cli.main run-script` accepts an arbitrary script body and dispatches
through the existing parser ‚Üí executor pipeline. Same execution mode
as a calendar event ‚Äî Smart or Plus auto-detected. No new permissions.

Risks: a user with shell access can already run anything via the CLI;
this endpoint exposes the same surface to the UI. Marker line in the
log: `[INFO] Sandbox run from menubar Recipes window`.

## Risks

| Risk | Mitigation |
|------|-----------|
| WKWebView in NSWindow needs explicit content layout | use `setContentView_` on a wrapper view; tested in PyObjC examples |
| Window controller leaked on quit | retain via `globals()["..."]`, terminate cleanly via NSApp.terminate |
| Recipe JSON schema drift | `schema_version` field; loader normalises missing keys |
| run-script PII leakage in logs | log a marker only, never the script body |
