# `_workspace/specs/` — accepted feature specs

These are the proposals that drove implementation of CalFlow features.
Each is short, opinionated, and frozen once accepted (changes to a shipped
spec are tracked in a NEW spec or an `addendum.md`).

Naming convention: `vX.Y.Z-<feature>.md`, lowercase-kebab-case.

Status flags inside each spec:
- `Status: draft` — under review, not yet accepted
- `Status: accepted` — user approved; implementation may begin
- `Status: shipped` — implementation merged + version tagged
- `Status: superseded by <newer-spec>.md` — keep file for history

This directory is **committed to git** so the design history of CalFlow
is recoverable forever.
