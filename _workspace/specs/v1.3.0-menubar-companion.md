---

## 1. Goal

A native macOS menubar app that replaces the CLI dashboard for everyday use.
Click the icon → see status, today's events, missed events (12 h), Recipes,
Settings, About, lifetime stats, Buy Coffee. Run-now any event with a play-icon.

Smart Mode + Plus Mode keep working untouched. The menubar is a **read-mostly
front-end** that talks to the same Python CLI everyone else already uses.

## 2. Architecture (decided)

```
   ┌──────────────────────────┐
   │  Menubar app (rumps)     │  cli/menubar.py
   │  ─────────────────────   │
   │  rumps icon + click      │
   │  → opens NSPanel         │
   │  → hosts WKWebView       │
   │     loads popover.html   │
   └────────────┬─────────────┘
                │   bridge: window.webkit.messageHandlers.cf.postMessage(...)
                ▼
   ┌──────────────────────────┐
   │  cli/main.py JSON API    │  read: status | upcoming | missed | stats
   │                          │  write: run-event <id> | pause | resume
   └────────────┬─────────────┘
                │
                ▼
   ┌──────────────────────────┐
   │  Existing pipeline       │  parser → resolver → executors
   │  (untouched)             │  + stats counter hook
   └──────────────────────────┘
```

**Why this stack:**
- `rumps` handles the menubar icon, click, and lifecycle in 30 LOC. Battle-tested.
- `WKWebView` lets us reuse the v6 mockup HTML verbatim — no rebuild in SwiftUI.
- The popover is HTML/CSS/JS that calls back into Python via the WebKit message
  handler. Python re-uses existing `cli.main` JSON endpoints. **Zero duplication.**
- Stays in Python — no Xcode, no separate signing identity, no second language.

**Why not pure rumps (text menu):** can't render the play-icon buttons, scrollable
missed list, or stats card from the mockup.

**Why not SwiftUI:** multi-day project, separate Xcode app, breaks the "one
language, one codebase" property of CalFlow.

## 4. Stats schema (decided)

**New file:** `data/stats.json` — separate from `state.json` because state is
time-pruned and size-capped; stats must be permanent and monotonic.

```json
{
  "first_run_date": "2024-07-28T14:32:01+00:00",
  "actions_run": 12546,
  "actions_failed": 87,
  "by_type": {
    "open_url": 6500,
    "open_profile": 1500,
    "arrange": 2000,
    "hide": 1000,
    "autofill": 1200,
    "focus": 300,
    "screenshot": 46
  },
  "schema_version": 1
}
```

`first_run_date` is set once on first write and never changed. `actions_run`
is the sum of `by_type` values. `by_type` keys match `ACTION_WEIGHTS` keys.

**Action weights (default — overridable via settings):**

| Key | Seconds | What it represents |
|-----|---------|-------------------|
| `open_url` | 5 | Open URL in default browser (Cmd+T + paste + Enter) |
| `open_profile` | 8 | Open URL in specific browser/profile (extra menu clicks) |
| `arrange` | 4 | Drag-snap a window to a half/grid cell |
| `hide` | 2 | Cmd+H or click away |
| `focus` | 1 | Cmd+Tab cycle |
| `autofill` | 8 | 1Password / iCloud Keychain lookup + paste |
| `screenshot` | 3 | Cmd+Shift+4 + drag + click |
| `wait` | 0 | Doesn't save time; don't count |

Total saved time = `Σ (by_type[k] × ACTION_WEIGHTS[k])`.

## 5. JSON API surface (cli.main)

All commands print one JSON object to stdout, exit 0 on success, non-zero on
fetch failure with `{"error": "..."}` on stderr.

```
python -m cli.main status   --json     (exists; v1.1.27)
python -m cli.main stats    --json     NEW
python -m cli.main upcoming --json     NEW   options: --hours N (default 24)
python -m cli.main missed   --json     NEW   options: --hours N (default 12)
python -m cli.main run-event <id> --json  NEW
python -m cli.main pause                NEW (calls launchctl unload — needs user approval gate, NOT auto-executed during build)
python -m cli.main resume               NEW (counterpart)
```

Schemas live in `docs/menubar.md`. Examples:

```json
// stats --json
{
  "first_run_date": "2024-07-28T14:32:01+00:00",
  "automations": 4182,
  "actions_run": 12546,
  "time_saved_seconds": 76410,
  "time_saved_human": "21h 14m",
  "by_type": { "open_url": 6500, ... }
}

// upcoming --json
{
  "events": [
    {"id": "abc", "title": "Standup", "start": "...", "seconds_until": 240,
     "mode": "smart", "calendar_id": "primary",
     "preview": "Opens Zoom in Chrome (Work)"}
  ]
}
```

without them (lazy-import inside `cli/menubar.py`).

## 8. Risks & mitigations

| Risk | Mitigation |
|------|-----------|
| WKWebView blocks file:// → cli.main calls | Use a JS bridge (`postMessage`) → Python; never call CLI from JS directly |
| rumps NSStatusBar lifecycle leaks on quit | Standard pattern: `rumps.quit_application()` on Cmd+Q |
| Stats counter hot-path slows executors | Batch writes — accumulate in memory, flush on event-end (every save_state) |
| First-run race on stats.json | Atomic write via tmp + os.replace (same pattern as state_manager) |
| `launchctl pause/resume` requires user approval per CLAUDE.md | `pause`/`resume` subcommands print the launchctl command for the user to run, do NOT execute it (matches existing `start`/`stop` policy) |
| WebKit screen on Apple Silicon vs Intel | `pyobjc-framework-WebKit` is universal; tested on both |

## 9. What this spec does NOT cover

- **First-run onboarding flow inside the menubar** — separate spec when needed.
- **Settings page implementation** — placeholder; clicking Settings opens
  `data/` in Finder and surfaces `settings.py` for now. Native UI in a follow-up.
- **About page update-check via GitHub Releases API** — placeholder; opens
  `https://github.com/.../releases/latest` for now.
- **Recipes (Playbook merge)** — placeholder; opens the `playbooks/` folder
  in Finder for now. The full editor is v1.4.x.
- **Native code-signing & Gatekeeper distribution** — out of scope; user
  runs `python -m cli.menubar` from terminal.

The above placeholders all open something useful; they don't break.

## 11. Migration

No migration required. First run creates `data/stats.json` and seeds
`first_run_date` with `datetime.utcnow()`. Existing users see "Since today
… 0 automations" until their first triggered event.
